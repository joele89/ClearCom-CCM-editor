  • This section allows you to display/configure the following elements

    • Name: click/touch here to give device a useful name.
    • Selected Role: the role currently associated with this device that controls the applied settings.
    • Model: model of the device.
    • Version: software and hardware version of device.

  • This Section allows you to manage administrative wireless options

    • OTA Registration: Opens the system up for Over The Air wireless registration of beltpacks.
    • OTA System Id: The wireless DECT identifier for the system. This must be unique to the operating area. Only change this value if absolutely required.
    • OTA PIN: A four-digit PIN code to allow FreeSpeak II beltpacks to register wirelessly with transceiver. The default PIN code is 0000, but Clear-Com strongly recommends that you change this for security reasons.
    • Administration PIN: A four-digit PIN that protects the FreeSpeak II beltpacks admin menu options.
    • AA Battery Type: Select between Alkaline and NiMH.
    • Audio RF Filter: Provides audio packet filtering, to address audio dropouts. Enable this mode when users are experiencing choppy audio on the system.
    • Role Sorting: Select the order roles are displayed on the beltpack when it connects.

  • This section allows you to configure the General Purpose Inputs and Outputs (GPIO). Up to 10 events can be trigger by one GPI and each GPO can be triggered by up to 10 trigger events.

    • GPI selector. Click on a GPI on the left. Then click the green + to select a source and destination. When the GPI is activated, a talk route will be created between the source and the destination.
      • Label: the user-friendly name for this input.
      • Event Source: When the input is activated this will be the talker for the new crosspoint. Please note that the microphone will not be automatically opened.
      • Event destination: When the input is activated this will be the listener for the new crosspoint.

    Example: a footswitch connected to GPI1 can make a crosspoint from the Station (source) to Channel 1 (destination).

    • GPO Selector: Click a GPO on the left. Then click the + to select a Key Target and On Behavior. When the action (Talk, Talk & Listen or Call) arrives at the target, the GPO is energised (activated).
      • Label: the user-friendly name for this output.
      • Key Target: whenever the route is made to this target the output is activated.
      • Key Behavior: the type of key press event to trigger the GPO.
        • Talk: either a Talk or a Talk & Force Listen key being depressed to the target
        • Talk & Listen: a beltpack with a Talk & Listen key to the target.
        • Call: a call signal to the target.

    Example: a stage announce light connected to GPO1 can be triggered when someone is talking to the stage announce port by setting Key Target to Stage Announce and the Key On Behavior to Talk.

  • This section allows you to configure Control Events from audio routes. When a source is routed to a destination (an Event) you must select one of three possible action types. Up to 10 Control Events are possible.

    Click + to add a new Control Event. Select the Source, Destination, and Action Type, then [Enter] key or click the blue check mark.

    Action Types:

    • Xpt: when the source is routed to the destination, a new crosspoint is made between a new source and a new destination. Please note that the microphone will not be automatically opened.
    • GPO: when the source is routed to the destination, the output relay specified in Action Details will be energized (activated). Once the route is disabled, the relay will de-energise. A relay can be set to be energized by more than one trigger or Control Event.
    • Call: when the source is routed to the destination, a call signal is sent to the destination.

    Note: if a Group or Channel is chosen as the source, then anyone talking to the Group or Channel will trigger the event.

  • This section allows you to change the CCM password of the host device (the device the browser is currently pointed at).

    This section allows you to change the CCM password of the Base Station.

  • This Section allows you to perform save and restore functionality on the device

    • Save: download a file containing current configuration details (Base Station, transceiver, beltpack, Role and connection settings). It can be used to restore your configuration if needed.
    • Restore: uploads and restores a previously saved system configuration file.

  • This section allows you to activate features using license codes.

  • This section allows you to activate features using license codes

    • Features: A summary of available features and their current activation status
    • Add license online (recommended): Clicking this will begin the process of activating features while the device is connected to the internet
    • Add license offline: Clicking this will begin the process of activating features if the device does not have an internet connection

  • This section allows you to upgrade the firmware of the device.

    • Click Select File to upload a firmware file from a local directory.
    • Once the file is uploaded, click Upgrade to initiate the upgrade process. All devices will automatically re-boot after upgrade.

  • This Section allows you to perform maintenance functions.

    • Reset to Default: use this button to reset the configuration to factory default.
    • Reboot: use this button to reboot the device.
    • Support Info: use this button to download the logs and information required to capture a fault condition.
    • Sync Clock: Sync the Base Station clock with the clock of the CCM client device. This ensures accuracy of the timestamp on the 'Support Info Files'.
